# Security Policy

## Reporting a vulnerability

**Do not open a public issue for security reports.**

Use [GitHub Security Advisories](../../security/advisories/new) to report privately.  
For critical issues, contact the brig·id maintainers via the org security contact.

## Scope

All brig·id repositories are in scope:
- cryptographic primitives and key management
- identity flows (WebAuthn, OIDC, DID)
- storage and encrypted vault
- deployment configuration

## Response SLA

| Severity | Acknowledgement | Remediation target |
|---|---|---|
| Critical | 48 h | 7 days |
| High | 72 h | 30 days |
| Medium/Low | 7 days | 90 days |

## Disclosure policy

We follow coordinated vulnerability disclosure (CVD).  
Public disclosure occurs after a fix is released or 90 days from report, whichever comes first.
